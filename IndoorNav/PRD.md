# PRD — IndoorNav

**Версия:** 1.0  
**Дата:** 04.03.2026  
**Статус:** Актуально

---

## 1. Обзор продукта

**IndoorNav** — мобильное/десктопное приложение для навигации внутри здания (университет). Строится на базе **.NET 9 MAUI** с SkiaSharp-рендерингом SVG-планов этажей, алгоритмом Дейкстры для прокладки маршрутов и встроенным редактором карты.

### Целевые платформы

| Платформа | Статус |
|-----------|--------|
| Windows 10/11 (unpackaged) | ✅ Основная |
| Android | ✅ Поддерживается |
| iOS / macOS Catalyst | ✅ Требует Mac + Xcode |

---

## 2. Цели и задачи

| Цель | Метрика успеха |
|------|----------------|
| Пользователь находит маршрут до нужной аудитории | ≤ 3 тапа от запуска |
| Студент видит, где проходит его текущая пара | Расписание подтягивается автоматически по группе |
| Администратор редактирует карту без перекомпиляции | Сохранение за ≤ 5 секунд через UI |
| ЧС-оповещение доходит до всех пользователей | Мгновенно после активации администратором |

---

## 3. Пользователи

### 3.1 Гость (Guest)
- Анонимный вход без пароля
- Полный доступ к навигации и QR-сканеру
- Нет доступа к расписанию групп и административным функциям

### 3.2 Студент (Student)
- Вход по логину/паролю
- Привязан к учебной группе и кафедре
- Видит текущую и следующую пару для своей группы
- Может строить маршруты до аудитории из расписания

### 3.3 Администратор (Admin)
- Полный доступ: редактор карты, управление пользователями, отделами, расписанием
- Активирует/деактивирует режим ЧС по зданиям
- Публикует обновления карты через GitHub Gist (опционально)

## 4. Функциональные требования

### 4.1 Авторизация

| ID | Требование |
|----|-----------|
| AUTH-1 | Вход по логину + паролю (SHA-256 хэш, хранение в `users.json`) |
| AUTH-2 | Вход без аккаунта (гость) одним кликом |
| AUTH-3 | По умолчанию существует учётная запись администратора `0` / `0` |
| AUTH-4 | Администратор меняет пароль любому студенту; студент меняет только свой |
| AUTH-5 | Сессия восстанавливается после перезапуска через `Preferences` |
| AUTH-6 | Разделение ролей: `Admin`, `Student`, `Guest` |

---

### 4.2 Навигация

| ID | Требование |
|----|-----------|
| NAV-1 | Выбор начальной и конечной точки через поиск по списку узлов |
| NAV-2 | Автодополнение поиска по имени, тегам, категории узла |
| NAV-3 | Построение маршрута алгоритмом Дейкстры через граф (`NavGraph`) |
| NAV-4 | Поддержка многоэтажных маршрутов через лестницы/лифты (`IsTransition`) |
| NAV-5 | Переходные узлы группируются по `TransitionGroupId`; штраф за смену этажа — 150 у.е. |
| NAV-6 | Пошаговые инструкции: текстовые подсказки + иконки; анимированный фокус на текущем шаге |
| NAV-7 | Счётчик шагов «N / M»; кнопки «Далее» / «Назад» |
| NAV-8 | Анимированный маршрут на карте (пунктир с бегущей точкой) |
| NAV-9 | Тап по узлу на карте открывает popup с именем, типом и кнопками «Отсюда» / «Сюда» |
| NAV-10 | Запасные выходы (`IsEvacuationExit`) исключаются из обычной маршрутизации — доступны только в режиме ЧС |
| NAV-11 | Обычные выходы (`IsExit`) также исключены из транзитных узлов маршрута |

---

### 4.3 QR-навигация

| ID | Требование |
|----|-----------|
| QR-1 | Каждый `IsQrAnchor`-узел имеет QR-код формата `indoornav://node/{guid}` |
| QR-2 | Сканирование QR устанавливает текущую позицию (начало маршрута) |
| QR-3 | Поддержка legacy URL: `indoornav-node:{guid}` и bare GUID |
| QR-4 | На Android/iOS — камера через ZXing.Net.Maui; на Windows — ввод текстом |
| QR-5 | Deep link обрабатывается через `DeepLinkService.RequestNode` |

---

### 4.4 Карта

| ID | Требование |
|----|-----------|
| MAP-1 | Рендер SVG плана этажа через SkiaSharp + Svg.Skia |
| MAP-2 | Наложение узлов, рёбер, границ комнат поверх SVG |
| MAP-3 | Пан (перетаскивание) + зум (mouse wheel / pinch); VSync-синхронизированный рендер |
| MAP-4 | Выбор здания и этажа через выпадающее меню в сайдбаре |
| MAP-5 | Разные визуальные стили для типов узлов: комната, переход, выход, огнетушитель, эвакуационный выход, QR-якорь |
| MAP-6 | Цвет узла настраивается (`NodeColorHex`); масштаб и метка — отдельно |
| MAP-7 | Огнетушители и запасные выходы скрыты в обычном режиме; видны при `ShowFireExtinguishers` (ЧС) |
| MAP-8 | Полигональные границы комнат (`Boundaries`) отображаются и редактируются |

---

### 4.5 Расписание

| ID | Требование |
|----|-----------|
| SCH-1 | Хранение расписания в `schedule.json` (день недели + временные слоты) |
| SCH-2 | Автоопределение текущей пары для комнаты или группы |
| SCH-3 | Студент видит свою текущую аудиторию и может проложить маршрут одним нажатием |
| SCH-4 | Администратор управляет расписанием: добавление/удаление/редактирование записей |
| SCH-5 | Просмотр расписания по комнатам или по группам |
| SCH-6 | Фильтрация расписания по группе |

---

### 4.6 Режим ЧС

| ID | Требование |
|----|-----------|
| EMRG-1 | Администратор активирует/деактивирует ЧС отдельно по каждому зданию |
| EMRG-2 | При активации все пользователи немедленно видят баннер с сообщением |
| EMRG-3 | В режиме ЧС строится маршрут до ближайшего выхода (`IsExit` или `IsEvacuationExit`) |
| EMRG-4 | Система запоминает, какие узлы заблокированы (`_blockedNodeIds`) — исключает их из роутинга |
| EMRG-5 | Пользователь может указать своё местоположение тапом при активации ЧС |
| EMRG-6 | Состояние ЧС сохраняется в `emergency_state.json` и восстанавливается после перезапуска |
| EMRG-7 | В ЧС-режиме на карте появляются огнетушители и запасные выходы |

## 5. Нефункциональные требования

| Категория | Требование |
|-----------|-----------|
| **Производительность** | Рендер карты синхронизирован с VSync (частота экрана); одного мастер-таймера достаточно для анимации маршрута + пана |
| **Надёжность** | Все данные хранятся локально; потеря сети не прерывает навигацию |
| **Безопасность** | Пароли хранятся как SHA-256 хэш; токены GitHub не публикуются в коде (поле пустое) |
| **Offline** | Полная работостособность без интернета; Gist-синхронизация — опциональная функция |
| **Расширяемость** | Новые здания добавляются через JSON + SVG без перекомпиляции |
| **UX** | Время от запуска до построения маршрута — не более 3 кликов |
| **Размер** | Приложение работает без MSIX-упаковки (`WindowsPackageType=None`) |

### 6.2 Слои

```

│            Service Layer                │
```

| Поле | Тип | Описание |

## 8. Пользовательские сценарии (User Flows)

### 8.1 Гость строит маршрут

```
Запуск → LoginPage → «Гость» → MainPage
  → Тап «Откуда» → Ввод поиска → Выбор узла
  → Тап «Куда» → Ввод поиска → Выбор узла
  → «Построить маршрут» → Анимированный маршрут на карте
  → Листание шагов инструкции
```

### 8.2 Студент находит свою аудиторию

```
LoginPage → «Пользователь» → логин/пароль → MainPage
  → Баннер «Сейчас: [аудитория]» → Тап → Маршрут проложен
```

### 8.3 QR-навигация

```
MainPage → кнопка QR → QrScanPage → сканирование кода
  → DeepLinkService.RequestNode → StartNode установлен → маршрут к цели
```

### 8.4 Режим ЧС

```
AdminPage → Управление ЧС → «Активировать» → все пользователи
  видят баннер → тап «Построить маршрут» → маршрут до
  ближайшего IsExit | IsEvacuationExit (заблокированные исключаются)
```

### 8.5 Администратор добавляет аудиторию

```
AdminPage → вкладка «Карта» → «Добавить узел»
  → тап на плане → ввод имени → сохранить
  → «Соединить» → тап на соседний узел → «Сохранить»
```

## 10. Ограничения и допущения

- Планы этажей хранятся как SVG-файлы в `Resources/Raw/SvgFloors/`; замена SVG требует пересборки
- GitHub Gist синхронизация (`AppConfig.cs`) отключена по умолчанию — поля пустые
- ZXing.Net.Maui подключается только на Android/iOS; на Windows QR сканируется вводом текста
- Координаты узлов привязаны к SVG-пространству 0–800 × 0–600; масштаб при пане/зуме — афинное преобразование

## 11. Известные ограничения текущей версии

| # | Описание |
